MySql

  1. data types

    • The data type is a guideline for SQL to understand what type of data is expected inside of each column,
      and it also identifies how SQL will interact with the stored data

  2. categories of commands

    • DDL
    • DML
    • TCL
    • DCL
    • DQL

  3. constraints

    • primary key
    • unique key
    • check
    • Not null
    • Foriegn key

  4. operators

    • An operator is a reserved word or a character that is used to query our database in a SQL expression

  5. Functions

    • They enhance the functionality and flexibility of SQL queries and enable developers to perform various operations on data

  6. joins

    • A JOIN clause is used to combine rows from two or more tables, based on a related column between them.

  7. clauses

    • clauses are specific parts of SQL statements that help you define, filter, or manipulate your database data

  8. views

    • VIEWS are virtual tables that do not store any data of their own but display data stored in other tables.

  9. sub queries

    • Subqueries in MySQL, also known as subquery or inner query, are queries that are embedded within another SQL query (the outer query).

  10. Index

    • In MySQL, an index is a database structure used to improve the speed of data retrieval operations on database tables

  11. stored functions

    • stored functions are a way to encapsulate a set of SQL statements into a reusable and modular block of code.
      These functions can accept parameters and return a single value

  12. stored procedures

    • A stored procedure is a precompiled collection of one or more SQL statements that can be stored in the database.
      It allows for modular code, reusability, and easier maintenance.

  13. cursors

    • In MySQL, a cursor is a database object used to retrieve and manipulate data from a result set.

  14. Exception handling

    • MySQL does not have built-in support for traditional exception handling like some other programming languages

  15. Triggers

    • A trigger is a set of SQL statements that are automatically executed when a specific event occurs in a table
